Compile Fossil SCM

一般將 Ubuntu 中的 fossil 設為 server, 配合 Stunnel 建立 https 介面, 而 Windows 中的 fossil.exe 則作為 client, 兩者可以透過 fossil version 查詢版次, 必須注意 server 與 client 的 Fossil SCM 版次是否相應, 以免無法使用新版本 Fossil SCM 可能新增的功能.

下載 Fossil SCM 最新版本原始碼:

從 https://www.fossil-scm.org/home/uv/download.html 選擇最新版本中的 history graph, 例如: Fossil SCM 2.18 版. 點選 ZIP archive 下載後解開原始碼壓縮檔案.

Ubuntu 操作系統:

若操作系統為 Ubuntu, 可以利用 sudo apt install libssl-dev 安裝所需的 OpenSSL 程式庫開發檔案.

接著利用 xterm 命令列進入 Fossil SCM 原始碼所在目錄後, 執行 ./configure 後, 再執行 make 就可以建立 fossil 可執行檔案. 只不過檔案大小約 17 MB, 若利用 ./configure --static 所建立的 fossil 則將近 22 MB.

Windows 64 位元操作系統:

以下將採用 MSYS2 執行編譯.

1. 從 Gnuwin32 coreutils 取得視窗環境下執行的 cat 與 grep.

2. 下載 Fossil SCM 最新版原始碼與上列 Ubuntu 相同.

3. 由於採用 MSYS2, 因此必須在 mingw64.exe 啟動後的介面, 以 pacman -S openssl-devel 安裝所需的 OpenSSL 程式庫開發檔案, 其中包括 libssl.a 與 libcrypto.a 等兩個靜態程式庫.

4. make.exe 檔案複製:

由於已經將 Y:\msys64_20210419\mingw64\bin\mingw32-make.exe 複製一份為 make.exe, 因此之後可以直接以 make 命令執行.

5. windres.exe 檔案複製:

另外必須要將 Y:\msys64_20210419\mingw64\bin\windres.exe 複製一份為 x86_64-w64-mingw32-windres.exe, 因為 win/Makefile.mingw 中會採用:

RCC = $(PREFIX)windres -I$(SRCDIR)

 轉換 Windows 原始碼中的 Resource files, 而 PREFIX 變數在下列將設定為 x86_64-w64-mingw32-.

6. 先編譯 compat/zlib:

cd compat/zlib

mingw32-make -f win32/Makefile.gcc

7. Makefile.mingw 設定：

MSYS2 所使用的 Makefile 位於 Fossil SCM 原始碼目錄中的 win/Makefile.mingw 檔案, 以 SciTE 編輯此一檔案時, 將第 22 行改為:

PREFIX = x86_64-w64-mingw32-

表示編譯完成的 fossil.exe 要在 x86_64-w64-mingw32 平台上執行.

第 65 行改為:

FOSSIL_ENABLE_SSL = 1

8. 從 http://gnuwin32.sourceforge.net/downlinks/make-dep-zip.php 下載 libintl3.dll 與 libiconv2.dll, 放至 y:\

表示編譯後的 fossil.exe 要能夠在 https 網路協定中使用.

9. OpenSSL 靜態程式庫檔案:

將 pacman -S openssl-devel 安裝後, 在Y:\msys64_20210419\mingw64\lib 目錄中的 libssl.a 與 libcrypto.a 檔案放入 Fossil SCM 原始碼目錄中.

10. 執行編譯與連結:

make -f win/Makefile.mingw LIB="-l:libssl.a -l:libcrypto.a -lgdi32 -lmingwex -lz -lws2_32 -municode"

此處採直接連結 libssl.a 與 libcrypto.a 的方式進行編譯, 完成後的 fossil.exe, 檔案大小約 8 MB.
