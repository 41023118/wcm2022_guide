w12_1a

cmsimde 網站的配置

請各組分別至電腦輔助設計室中帶有 3 個開機磁區的電腦設定一台 W10 虛擬主機, 將各分組的動態與靜態網站以可攜 Python, nginx, stunnel, NSSM 與 waitress 進行配置.

  1. Python 加上 flask flask_cors bs4 lxml pelican markdown leo pyopenssl 等模組, 可採用 self-signed certificate https 啟動 cmsimde 所建立的動態網頁系統.
  2. nginx 可以採用 self-signed certificate https 協定伺服 cmsimde 所建立的靜態網頁系統.
  3. stunnel 可以配合內部 http nginx 網站, 代理外部 request 的 https 協定連結.
  4. NSSM 可以用來將採 cmsimde 建立的網站系統設定為 Windows 的服務.
  5. waitress 可以用來將採 cmsimde 建立的動態網站系統, 以 WSGI 模式啟動.

Fossil SCM server 配置

Fossil SCM server 乃將 Fossil SCM 配置在 WWW 介面下, 使用者透過瀏覽器登入系統後可以在自行建立的分散式版次管理系統下, 配置 cmsimde 所建立的靜態網頁系統, 也可以設法透過 Oauth 登入後同時編輯動態網頁, 並登入 Fossil SCM 使用網際 Forum 與 Chat 等協同工具.

OpenSSL for Windows x64

網際協同內容管理

  1. 各分組的 wcm2022ag1 位於第一組組長的倉儲帳號下, 請組長將所有組員設為該分組倉儲的協同管理者
  2. 各組組員登入各自的 Github 帳號, fork 組長帳號下的分組倉儲, 設定 Github Pages 後, 以逆向 Pull Requests 直接在網際介面, 讓各自帳號下的分組倉儲內容與組長帳號下的分組倉儲內容保持同步.

將 Brython 程式環境配置到各組的分組倉儲

配置 Brython 程式環境的步驟:

  1. 導入 brython.js 解譯程式庫
  2. 導入 brython_stdlib 標準程式庫
  3. 在頁面中啟動 Brython
  4. 在頁面中編寫 Brython 程式

採協同模式建立各分組的 Tetris 網際程式

將 Tetris.py Brython 程式碼存在各分組倉儲的 downloads 目錄下, 由各學員採協同設計模式進行程式註解與開發後, 透過 Github Pages 在 Brython 環境的 URL 連結下, 以 ?src=Tetris.py_的_URL 呼叫執行.

範例: 原始的 tetris_game.py: https://mde.tw/wcm2022_guide/content/run.html?src=https://mde.tw/wcm2022_guide/downloads/tetris_game.py

當多人同時維護組長帳號下分組倉儲中的 tetris_game.py 時, 各組員在近端編輯之前必須要先設法讓程式版本與遠端倉儲同步 (git pull), 改版後進行提交推送時, 若當下遠端程式內容已經被其他組員改版, 就必須再設法利用 git pull 拉回資料後, 設法解決衝突後, 才能執行 git push.
